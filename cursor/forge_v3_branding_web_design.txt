



Am studiat pagina ta /coming-soon și documentația tehnică a proiectului PROMPTFORGE™.
Ai deja un design minimal bun (dark, glass-effect, grid simplu). Dacă îl amplifici corect, site-ul nu va părea „ieftin”, ci solid și lucrat mult timp.

Cerințele de webdesign frontend și UI/UX pe care să le scrii tehnic (ca backlog pentru designer/frontend) arată așa:

1. Consistență vizuală (Glassmorphism industrial)

Folosește .glass-effect și .glass-strong pe toate cardurile (hero, form, proof-bar).

Menține contrastul ridicat (fundal #0A0A0A, text #ECFEFF, accent teal/crimson).

Aplică glow discret doar pe CTA-uri primare și stări active, niciodată pe text de corp.

2. Tipografie și ierarhie

Headinguri Montserrat (600/700), body Open Sans.

Păstrează spațiere strictă: 4 / 8 / 12 / 16 / 24 / 32 px (canon PROMPTFORGE™).

Adaugă „proof-bar” sub CTA (3 bullets scurte, tehnice: ex. „TTA < 60s • Score ≥ 80 • Export .md/.pdf/.json”).

3. Micro-interacțiuni „scumpe”

Animații subtile (fade-in, shimmer pe buton, slide-up pe form). Durată 180–220ms.

Hover pe butoane: lumină radială mică (glitch neon).

Formularele trebuie să aibă feedback instant: border teal on-focus + ring accent.

4. Layout & grid

Container max-width: 1240px, grid 12 col, gutter 16px.

Keep it minimal: 1–2 blocuri full-bleed (hero + form), restul spațiu negativ.

Evită „secțiuni suplimentare” – doar CTA central + waitlist.

5. Elemente de încredere

Adaugă sub form: „Zero spam. Doar notificări de lansare.”

Integrează badge-uri vector (V1–V7) mici, colorate doar pe text (ex. text-blue-400), nu pe carduri.

Footer minimal: © PromptForge + link „Privacy/Terms”.

6. Coerență UX

CTA principal unic (buton mare neon teal „Join the Waitlist”).

Elimină dublările – un singur punct de acțiune clar.

Focus vizibil la tab/keyboard nav (accesibilitate by Radix).

7. Cod & guardrails

Blochează linter sub score 80 la texte (Hero, CTA).

Fără „magie, ușor, inspirațional” – lexic interzis în brand.

Tot conținutul să treacă prin validatePromptStructure (titlu, sub, CTA, proof-bar).

🔑 Pe scurt: nu schimbi designul, ci îl „întărești” prin consistență (glass+grid), micro-detalii (shimmer, glow, focus states), și prin „proof-bar” tehnic. Astfel, vizitatorul simte că e o fabrică serioasă, nu un landing făcut rapid.






Punctul 1 – Consistență vizuală (Glassmorphism industrial), formulată ca set de cerințe tehnice UI/UX executabile pentru echipa de design/frontend:

🧊 1. CONSISTENȚĂ VIZUALĂ – GLASSMORPHISM INDUSTRIAL
🎯 Obiectiv:

Păstrează un aspect tehnologic, rece, solid – fără efecte „drăguțe” sau elemente generice. Aplică tratamentul glassmorphic funcțional, inspirat din brandingul PromptForge™. Toate componentele vizuale trebuie să simuleze materiale „industriale digitale” – sticlă groasă, metal anodizat, accente luminoase de precizie.

✅ Reguli tehnice:
a) Carduri și suprafețe flotante

Aplică clasa .glass-effect pe toate componentele care definesc blocuri principale (hero, formulare, CTA-uri secundare, secțiuni de proof).

.glass-effect {
  background-color: rgba(255,255,255,0.08);
  backdrop-filter: blur(12px);
  border: 1px solid rgba(255,255,255,0.15);
  border-radius: 12px;
}


Folosește .glass-strong pentru elemente critice (ex. overlay modal, form de submit).

.glass-strong {
  background-color: rgba(255,255,255,0.12);
  backdrop-filter: blur(16px);
  border: 1.5px solid rgba(255,255,255,0.2);
}

b) Culori și contrast

Fundal global: #0A0A0A (nu #000, nu gradient).

Text principal: #ECFEFF (white blueish – citibilitate maximă pe negru).

Accent primar: #164E63 la stare normală → #0891B2 la hover (teal rece, tehnic).

Accent critic: #BE123C (nu roșu cald — crimson cu semnificație operațională, nu decorativă).

c) Glow: cu măsură, doar pe funcțional

Folosește .glow-primary și .glow-accent DOAR pentru:

butonul principal din Hero

CTA-uri de înscriere

elemente hover state din formular

Efectul glow se aplică subtil, cu halo difuz + spread scurt (ex: box-shadow: 0 0 8px #0891B2).

d) Elemente nepermise

✘ Fără efecte glossy, gradient în litere sau corner rounding exagerat.

✘ Fără imagini de background, pictograme decorative sau mascote.

✘ Nu colora cardurile în funcție de vectori — vector colors doar pe <Badge> (ex: text-blue-400, nu bg-blue-400).

📐 Exemple de aplicare:
Hero Section
<div className="glass-effect glow-primary p-12 rounded-xl max-w-3xl mx-auto">
  <h1 className="text-3xl font-bold text-foreground">PromptForge™ is coming.</h1>
  <p className="text-muted mt-4">Join the waitlist. Build prompt systems, not throwaway lines.</p>
  <button className="btn btn-primary mt-6 glow-primary">Join the Waitlist</button>
</div>

Proof-bar sub CTA
<div className="glass-strong mt-6 text-sm text-muted p-4 rounded-md flex justify-between">
  <span>TTA &lt; 60s</span>
  <span>AI Score ≥ 80</span>
  <span>Export .md/.pdf/.json</span>
</div>

🧪 Acceptance criteria pentru Dev/QA:

 Toate componentele majore (hero, carduri, form) au .glass-effect sau .glass-strong.

 Butonul principal are glow activ doar în hover/focus (nu permanent).

 Vectorii colorați (V1–V7) apar doar în Badge-uri, nu pe fundaluri sau butoane.

 Nu există niciun gradient, emoji, sau decorațiuni în afara codului din branding.

 Toate codurile de culoare respectă schema PromptForge™ (Teal/Crimson strict definite).






✍️ 2. TIPOGRAFIE ȘI IERARHIE – PRECIZIE INDUSTRIALĂ
🎯 Obiectiv:

Creezi o ierarhie clară, predictibilă, ușor de scanat vizual. Stilul PromptForge™ este ingineresc, nu editorial: fără litere mari de dragul decorului, fără sloganuri poetice. Fiecare rând de text trebuie să pară parte dintr-un sistem de documentație tehnologică, nu dintr-o reclamă.

✅ Reguli tehnice:
a) Fonturi (already implemented în layout.tsx)

Headings: Montserrat, greutate 600 / 700 / 900, variabila --font-heading

Text/UI body: Open Sans, regular/medium, variabila --font-sans

Aplicare:

<h1 className="font-heading font-bold text-3xl text-foreground">PromptForge™ is coming</h1>
<p className="font-sans text-muted mt-4">Join the waitlist to access the semantic forge.</p>

b) Dimensiuni & ierarhie logică
Nivel	Font-size	Greutate	Spațiere verticală	Observații
H1 (Hero)	text-4xl / text-5xl	font-extrabold	mb-4	Max 1 per pagină
H2	text-2xl	font-bold	mt-12 mb-2	Secțiuni
H3	text-xl	font-semibold	mt-8 mb-1	Subsecțiuni
Paragraph	text-base	font-normal	mt-2 mb-4	Maxim 80c/linii
Caption	text-sm / text-xs	text-muted	mt-1	Pentru form/help

Nu folosi text-center decât în hero sau în mod explicit justificat.
Textul funcțional trebuie să rămână aliniat la stânga.

c) Spacing & padding

Regula strictă: 4 / 8 / 12 / 16 / 24 / 32 / 48 (doar aceste valori pentru p- și m-)

În componenta Hero: p-12 + gap-y-4

Evită spațiere neregulată sau diferențe de baseline.

d) Proof-bar (standard branding)

Sub orice CTA trebuie să apară o bară de încredere cu 2–3 bullets scurte, ex:

<div className="glass-strong mt-6 px-4 py-2 text-sm text-muted rounded-md flex justify-between">
  <span>TTA &lt; 60s</span>
  <span>AI Score ≥ 80</span>
  <span>Export .pdf/.json/.md</span>
</div>

e) Lexic permis

„Generează”, „Configurează”, „Exportă”, „Scor ≥80”, „Forjează”

Fără cuvinte ca: „magie”, „ușor”, „pasiune”, „descoperă frumusețea AI‑ului” (vezi lexicul interzis din Brand OS)

🎨 Exemplu concret (hero + proof-bar)
<section className="glass-effect max-w-2xl mx-auto p-12 mt-24 rounded-xl">

  </div>
</section>

🧪 Acceptance criteria pentru QA:

 Toate heading-urile folosesc Montserrat și respectă text-4xl/2xl/xl, fără exces de bold sau uppercase.

 Textul body e 100% Open Sans (text-base), fără alte fonturi sau scale personalizate.

 Fiecare paragraf are max. 80 caractere/linii și spațiere mt-2 mb-4.

⚙️ 3. MICRO‑INTERACȚIUNI „SCUMPE” – ANIMAȚII SUBTILE CU VALOARE PERCEPUTĂ MARE
🎯 Obiectiv:

Simulează un produs matur, costisitor, lucrat cu grijă.
Publicul tău (mentori, fondatori, dezvoltatori) detectează imediat elementele făcute „în grabă”.
Valoarea percepută se transmite prin detalii minuscule, executate cu precizie, control și intenție – nu prin efecte spectaculoase.

✅ Reguli și specificații tehnice
a) Durată animații

Toate tranzițiile să se încadreze între 180ms – 220ms

Excepție: shimmer pe schelet (poate dura 2.5s loop)

transition-duration: 180ms;
transition-timing-function: ease-out;

b) Efecte permise (din app/globals.css)
Efect	Clasă	Aplicație
Fade in	.animate-fade-in	Texte, butoane, badge-uri
Slide up	.animate-slide-up	Componente la intrare (ex: proof-bar, carduri)
Shimmer	.loading-shimmer	Carduri/skeleton loading
Bounce subtil	.animate-bounce-subtle	CTA-uri doar la hover
Glow	.glow-primary, .glow-accent	Doar pe butoane & badge-uri selectate
c) Hover states — rafinament, nu gimnastică

Pe button.btn-primary:

transform: translateY(-1px) scale(1.01);
box-shadow: 0 0 8px #0891B2;
background-color: #0891B2;
transition: all 180ms ease-out;


Pe carduri .glass-effect:

Ușoară creștere a luminozității (brightness(1.05))

Nu folosi scale(1.05) exagerat — maxim scale(1.01)

d) Focus states – accesibil, dar tech

Input: border rgba(22,78,99,0.5) + halo discret teal

Button: outline + ring-2 ring-teal-500 ring-offset-2

<input
  className="focus:outline-none focus:ring-2 focus:ring-offset-2 focus:ring-teal-600"
/>

e) Loading skeletons – rafinament

Folosește .loading-shimmer pentru carduri și module încărcate:

@keyframes shimmer {
  0% { background-position: -500px 0; }
  100% { background-position: 500px 0; }
}
.loading-shimmer {
  background: linear-gradient(to right, #0a0a0a 4%, #1f2937 25%, #0a0a0a 36%);
  background-size: 1000px 100%;
  animation: shimmer 2s infinite linear;
}

f) Animații pe secțiuni (entry)

Apare Hero → fade-in + slide-up (delay 100ms)

Apare Form → delay-300 cu animate-slide-up

Apare Proof-bar → delay-500 cu fade-in

Badge-urile: opacity-0 → 100% cu delay staggered

🛠️ Exemple de implementare (Tailwind + CSS)
<button className="btn btn-primary glow-primary transform transition-transform duration-200 ease-out hover:-translate-y-[1px] hover:scale-105">
  Join the Waitlist
</button>

<div className="glass-effect p-6 animate-fade-in animate-slide-up delay-300">
  <p className="text-muted text-sm">Secure your spot. No spam.</p>
</div>

<div className="h-48 w-full rounded-md loading-shimmer mt-4" />

🚫 Efecte interzise
Efect	Motiv
bounce, wobble, pulse dramatice	Arată „ieftin”, în stil template
Gradient de tip „rainbow”	Rupe identitatea industrială
Animații necontrolate infinite	Distrag atenția / par jucăușe
Animație la fiecare scroll	Încarcă inutil pagina, e obositor
Flash, scale mari (>1.1)	Arată ca site de e-book
🧪 Acceptance criteria pentru QA:

 Toate butoanele au tranziție între 180–220ms și hover:scale(1.01) max.

 Cardurile .glass-effect au hover:brightness(105%) și efect fade-in sau slide-up.

 Nu există niciun hover fără transition.

 Componentele de tip form sau badge au focus-visible:ring-* corect.

 Skeletons folosesc .loading-shimmer, cu ritm de animație constant și neagresiv.

 Nu există nicio animație „gimnastică” (bounce, rotate, jiggle).

 Nu există animații pe scroll sau efecte infinite care nu sunt shimmer.

 Linterul nu raportează nicio clasă Tailwind nerecunoscută sau conflict stilistic.




📐 4. LAYOUT & GRID — STRICȚIE INDUSTRIALĂ
🎯 Obiectiv:

Folosește un layout industrial, rigid, aerisit, cu spațiere calculată și conținut minim. Nu este un site de storytelling vizual, ci o linie de producție semantică.
Designul trebuie să comunice: „aici totul este calibrat și măsurat”.

✅ Reguli tehnice:
a) Container principal

Folosește max-width: 1240px (max-w-screen-xl mx-auto)

Padding global lateral: px-4 sm:px-6 lg:px-8

b) Grid

Grid standard 12 coloane

<div className="grid grid-cols-12 gap-4">
  <div className="col-span-12 sm:col-span-8 sm:col-start-3">...</div>
</div>


Gutter constant: gap-4 (16px)

Nu folosi offseturi hardcode — folosește col-start, col-span

c) Compoziție

Doar 1–2 blocuri full-bleed pe verticală:

Hero (H1 + Sub + CTA)

Formulă de înscriere (cu proof-bar)

Tot restul este spațiu negativ controlat (mt-48, py-32, pb-60)
Designul e respirabil, nu dens.

d) Excluderi

✘ Fără secțiuni intermediare de tip „features”, „testimoniale”, „de ce noi”.

✘ Fără imagini, icon-uri, divizări decorative.

🧪 Acceptance criteria pentru QA:

 Container global este max-w-screen-xl mx-auto px-4.

 Există un singur bloc Hero și un singur bloc Form.

 Nu există mai mult de 2 secțiuni verticale în structura paginii.

 Layoutul folosește grid-cols-12 cu gap-4, nu flex cu margin-left.

 Toate coloanele folosesc Tailwind col-span-*, col-start-*.

🔄 6. COERENȚĂ UX — UNITATE FUNCȚIONALĂ, FĂRĂ EXCES
🎯 Obiectiv:

Asigură-te că fiecare element din interfață are un singur rol clar și că niciun pas nu e dublat.
Nu există loc pentru redundanțe, estetici „de umplutură” sau experiențe confuze.
PromptForge™ operează ca o linie de producție – eficiență, predictibilitate, precizie.

✅ Reguli de implementare UX
1. ✅ CTA PRINCIPAL UNIC
✦ Rol:

Dirijează toată acțiunea utilizatorului către un singur punct decisiv (ex: înscriere waitlist)

Orice altă cale → disonanță → scăderea ratei de conversie

✦ Specificație:
<button className="btn btn-primary glow-primary w-full max-w-md mx-auto">
  Join the Waitlist
</button>

✦ Cerințe:

Butonul să fie unicat în pagină (nu repetat sus și jos)

Aliniere centrală, spațiu în jur (min. mt-8, max mb-16)

Buton size=lg, font Montserrat font-semibold, animație glow-primary la hover

✦ Interdicții:

✘ Fără „Secondary CTA” în pagina coming soon

✘ Fără „Learn more”, „Read docs”, „Vezi ce urmează” — doar CTA > conversie

2. ✅ FOCUS VIZIBIL & ACCESIBILITATE (A11Y)
✦ Rol:

Orice utilizator (keyboard/tab) trebuie să poată parcurge interfața fără mouse

Focusul trebuie să fie vizibil, dar discret

✦ Standard Tailwind + Radix:
<input
  className="focus:outline-none focus:ring-2 focus:ring-offset-2 focus:ring-teal-500"
/>

✦ Cerințe:

Toate inputurile și butoanele folosesc focus:ring-* + ring-offset-2

Nu ascunde outline implicit

Verifică navigarea completă prin Tab → Space → Enter

✦ Test minimal:

Tab → ajunge pe form

Space → deschide dropdown (dacă e)

Enter → trimite formularul

Shift+Tab → se întoarce corect

3. ✅ ELIMINĂ DUBLĂRI DE ACȚIUNE
✦ Obiectiv:

Nu lăsa utilizatorul să aleagă „între două butoane care fac același lucru”

Confuzia mică → scăderea autorității percepute

✦ Exemplu corect:

🎯 Obiectiv:

Transformă conținutul vizual și textual al paginii în artefacte validabile, nu simple texte decorative.

✅ 3 Straturi de validare
1. ✅ Linter semantic — blocare sub scor 80
✦ Rol:

Orice text (H1, subheading, CTA, mesaj formular, footer) trebuie să fie valid semantico-retoric.

Se validează cu validatePromptStructure() + scorul generat de evaluatePrompt() (claritate, execuție, business_fit, ambiguitate)

✦ Regulă hard:
if (scoreTotal < 80) {
  throw new Error("PromptForge Linter: text invalid — scor sub 80/100.");
}

✦ Secțiuni validate:

H1 (titlul)

Subheading

Buton CTA

Proof-bar

Footer

2. ✅ Lexic permis / lexic interzis
✦ Rol:

Menține vocea PromptForge™ – rece, tehnică, industrială.

✦ Cuvinte PERMISE:

Generează, Forjează, Rulează, Configurează, Exportă, Score ≥ 80, Artefact, Prompt industrial, Sistem, 7‑D, Guardrail, Telemetrie

✦ Cuvinte INTERZISE (blocare):
const BANNED = ["ușor", "magic", "inspirațional", "minunat", "pasiune", "îți schimbă viața", "descoperă frumusețea"];


❌ Exemplu invalid:

„PromptForge™ este un tool magic care îți schimbă viața.”
⛔ Blocare: magic, îți schimbă viața

✅ Exemplu corect:

„Forjezi prompturi scalabile cu scor ≥ 80. În mai puțin de 60 secunde.”

✦ Implementare:
const hasBannedWords = text => BANNED.some(word => text.toLowerCase().includes(word));
if (hasBannedWords(input)) throw new Error("PromptForge Guardrail: lexic interzis.");

3. ✅ Validarea structurii — secțiuni necesare
✦ Validare structurală obligatorie (ca la un prompt):

Toate textele publice trebuie să conțină minim:

Secțiune	Conținut	Validat de
H1	Titlu clar, fără metaforă	prompt.h1
Subheading	Context direct, fără emoții	prompt.context
CTA	Acțiune unică, imperativ	prompt.cta
Proof-bar	KPI verificabili, scurți	prompt.metrics
Footer	Legal + referință clară	prompt.footer_legal
✦ Fără una dintre ele → execuție invalidă
🧪 Acceptance Criteria pentru QA:
✅ Linter semantic:

 Toate textele publice au scor ≥ 80 la evaluatePrompt().

 Dacă scorul este < 80, butonul CTA este dezactivat + mesaj vizibil în console/log.

✅ Lexic:

 Niciun text nu conține cuvintele interzise din lista BANNED.

 Codul returnează eroare la build/deploy dacă detectează termeni interziși.

✅ Structură:

 Există exact 1 H1, 1 subheading, 1 CTA, 1 proof-bar, 1 footer.

 Dacă lipsește una → ecran de alertă „prompt incomplet”.

function validatePageTexts(texts: string[]) {
  const banned = texts.find(t => hasBannedWords(t));
  if (banned) throw new Error(`Lexic interzis: ${banned}`);
  texts.forEach(text => {
    const score = evaluatePrompt(text);
    if (score.total < 80) throw new Error(`Scor sub 80: ${score.total}`);
  });
  const hasAllSections = checkSectionPresence(texts);
  if (!hasAllSections) throw new Error("Structură incompletă (H1, CTA, Proof-bar etc.)");
}
